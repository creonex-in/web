Read the following files in order before implementing
or making any architectural decision:

1. `.claude/project-overview.md` — product definition, goals, features, and scope
2. `.claude/architecture.md` — system structure, boundaries, rendering strategy, and routing
3. `.claude/style-guide.md` — brand colors, typography, spacing, and design tokens
4. `.claude/ui-context.md` — theme tokens, component patterns, and Tailwind utilities
5. `.claude/code-standards.md` — TypeScript, React, Next.js, and Tailwind conventions
6. `.claude/progress-tracker.md` — current phase, completed work, open questions, and next steps

Update `.claude/progress-tracker.md` after each meaningful implementation change.

If implementation changes the architecture, scope, or standards documented in the context
files, update the relevant file before continuing.
